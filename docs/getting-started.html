Quick Start

Get Started with Resonance

From zero to your first match in under five minutes. No account, no registration, no cloud dependency.

Option 1

Download the Desktop App

The easiest way to get started. Download, unzip, and double-click. Opens in your browser automatically.

After downloading

macOS: Unzip, then right-click the binary → Open (required first time for unsigned apps).
Linux: tar xzf resonance-linux-x64.tar.gz && chmod +x resonance-linux-x64 && ./resonance-linux-x64
Windows: Unzip, then double-click resonance-win-x64.exe.

Your browser will open to http://localhost:3000 where you can create your identity and start publishing needs and offers.

Or install with Homebrew (macOS/Linux)

# One-time setup brew tap Planetary-e/resonance # Install and run brew install resonance resonance

After tapping, future installs and upgrades are just brew install resonance / brew upgrade resonance.

Option 2 — Developers

What You Need

⚙

Node.js 20+

Resonance requires Node.js version 20 or later. Check with node --version.

npm

Walkthrough

Step by Step

Follow these six steps to install Resonance, create your identity, publish needs and offers, find matches, and connect with peers — all from your terminal.

Step 1: Install

Install the Resonance CLI globally via npm:

# Install globally npm install -g @planetary/resonance # Verify installation resonance --version

Step 2: Initialize Your Node

Create your local identity and configuration. This generates an Ed25519 keypair (did:key) and stores it locally. No server is contacted.

resonance init
What happens

A did:key identity (Ed25519) is generated and stored in ~/.resonance/. This is your cryptographic identity. No email, phone, or username is needed. You can regenerate it at any time.

Step 3: Publish a Need or Offer

Describe what you need or what you offer in plain language. The CLI embeds the text locally using Nomic-embed-text (768 dimensions), applies Laplace noise for differential privacy, and publishes the protected vector to the relay mesh.

# Publish a need resonance publish --type need "Looking for a React developer with WebRTC experience" # Publish an offer resonance publish --type offer "Senior frontend engineer, 5 years React, real-time video" # Adjust privacy level (low=5.0, medium=1.0, high=0.1) resonance publish --type need --epsilon 0.1 "Sensitive request with maximum privacy"

Step 4: Search for Matches

Query the relay mesh for complementary matches. Results are returned with similarity scores. The relay threshold is 0.50; confirmation threshold is 0.55.

resonance search "React developer with WebRTC experience"

Results include match scores and category-level information. No personal details are revealed until you choose to connect.

Step 5: Connect with a Match

When you find a match above the confirmation threshold (0.55), initiate a direct encrypted channel. A Diffie-Hellman key exchange (X25519) derives a shared secret for end-to-end encryption.

# Connect to a matched peer by their DID resonance connect <peer-did>

Step 6: Progressive Disclosure

Send messages through the encrypted channel. You control what you reveal and when. The protocol supports three disclosure levels: category, detail, and contact.

# Open a channel and send a message resonance channel <peer-did> "Hi! I saw we matched on frontend development. Can you tell me more about the project?"
E2E Encryption

All channel messages are encrypted with XSalsa20-Poly1305 using a shared secret derived from X25519 Diffie-Hellman key exchange. Neither relays nor any third party can read your messages.

Reference

CLI Commands

The Resonance CLI provides 8 commands that cover the full lifecycle from identity creation to encrypted communication.

Command Description Key Options
init Initialize a new node: generate Ed25519 keypair and local config --relay <url> to set default relay
publish Publish a need or offer to the relay mesh --type need|offer, --epsilon <value>, --category <cat>
search Search for complementary matches on the network --limit <n>, --threshold <score>
matches List current matches and their status --format json|table
connect Initiate a DH key exchange and open a channel with a peer <peer-did>
channel Send or receive E2E encrypted messages on an open channel <peer-did>, <message>
status Show node status: identity, connected relays, active publications --verbose
serve Start the node as a long-running process (relay or personal node) --port <n>, --relay
Power Users

Running a Relay

Relays are lightweight, stateless nodes that facilitate matching between peers. Anyone can run one. They never see raw text — only noisy vectors and encrypted metadata.

# Initialize a relay node resonance init --relay # Start the relay resonance serve --relay --port 4444
Relay Architecture

Relays store only differentially-private vectors and ephemeral routing metadata. They cannot reconstruct the original text, identify users beyond their did:key, or read channel messages. If your relay goes down, peers automatically route through other relays in the mesh.

Development

Dev Cluster

For development and testing, spin up a local cluster with multiple nodes and a relay using the included script.

# Clone the repository git clone https://github.com/Planetary-e/resonance.git cd resonance # Install dependencies npm install # Start a local dev cluster (relay + 2 nodes) bash scripts/dev-cluster.sh

The dev cluster script starts a local relay and two peer nodes, publishes sample needs and offers, and demonstrates the full matching flow. It's the fastest way to see the protocol in action.

Test Suite

Run npm test to execute all 119 tests across the core library, protocol layer, and privacy module. Run npm run eval to generate evaluation reports with all 35 metrics.
