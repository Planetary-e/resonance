Open Source

Contributing

Resonance is open source and welcomes contributions. Whether you are fixing a bug, adding a benchmark, or improving documentation, here is everything you need to get started.

Setup

Development Setup

Get the project running locally in a few steps.

Prerequisites

Node.js 20+

The project uses modern ESM features and requires Node.js 20 or later. Check your version with node --version.

npm

Package management via npm. The monorepo uses npm workspaces to link packages together.

Git

Standard git workflow. Fork the repository, create a feature branch, submit a pull request.

Clone and Install

# Clone the repository git clone https://github.com/planetary/resonance.git cd resonance # Install all dependencies (monorepo-wide) npm install # Verify everything works npx vitest run
First Run

The first time you run tests or eval, the embedding model (~90MB) will be downloaded and cached. Subsequent runs start in under a second.

Structure

Project Structure

The monorepo is organized into three packages, each with a focused responsibility.

resonance/ packages/ core/ // The heart of the system src/ types.ts // Shared types, constants, config crypto.ts // Identity, signing, encryption perturbation.ts // Laplace noise / differential privacy embedding.ts // HuggingFace transformer embeddings vectors.ts // Vector math (cosine sim, normalize) protocol.ts // Wire protocol messages __tests__/ // Unit tests for core relay/ // Relay server src/ matching-index.ts // Separate HNSW for needs/offers relay-server.ts // WebSocket server + admin API __tests__/ // Unit tests for relay eval/ // Evaluation and benchmarks src/ benchmarks/ // Individual benchmark modules bootstrap.ts embedding.ts perturbation.ts hnsw.ts recall.ts rewriting.ts matching-index.ts cross-language.ts latency.ts run.ts // Main runner + report generator docs/ // Documentation site (this site) assets/style.css // Shared stylesheet evals/ // Auto-generated eval reports
Testing

Running Tests

Unit Tests

The project has 74 unit tests covering crypto, perturbation, vectors, protocol, and matching.

# Run all tests npx vitest run # Run tests in watch mode npx vitest # Run tests for a specific package npx vitest run --filter core npx vitest run --filter relay

Evaluation Suite

The eval suite is separate from unit tests. It runs real benchmarks with real data and generates reports.

# Quick eval: 10K vectors, ~2 minutes npm run eval:quick # Full eval: 100K vectors, ~15 minutes npm run eval # Reports are saved to docs/evals/eval-YYYY-MM-DD-HH-MM-SS.md

Patterns

Key Patterns & Dependencies

Important patterns and libraries used throughout the codebase.

Pattern / Dependency Used For Notes
Float32Array All vector storage Memory-efficient (3KB per 768-dim vector). Used everywhere vectors are passed.
tweetnacl Cryptography Ed25519 signing, X25519 key exchange, NaCl box/secretbox. Zero dependencies, audited.
@huggingface/transformers Embeddings Local transformer inference. No API calls. Model downloaded on first use (~90MB).
hnswlib-node Approximate nearest neighbor Native HNSW implementation. Used in the relay's MatchingIndex for fast vector search.
vitest Testing Fast ESM-native test runner. Compatible with Jest API.
TypeScript strict Type safety All packages use strict mode. No any types. All functions are typed.
ESM Module system Pure ESM throughout. No CommonJS. "type": "module" in all package.json files.
Benchmarks

Adding a Benchmark

To add a new benchmark to the eval suite, follow this pattern.

1. Create the Benchmark File

Create a new file in packages/eval/src/benchmarks/:

// packages/eval/src/benchmarks/my-benchmark.ts import type { BenchmarkResult } from '../types.js'; export async function runMyBenchmark(): Promise<BenchmarkResult[]> { const results: BenchmarkResult[] = []; // Your benchmark logic here... results.push({ metric: 'My metric name', target: '>0.50', // or 'info' for tracking-only actual: '0.72', pass: true, // or null for info metrics }); return results; }

2. Wire into run.ts

Import and call your benchmark in the main runner:

// packages/eval/src/run.ts import { runMyBenchmark } from './benchmarks/my-benchmark.js'; // Add to the benchmark list const benchmarks = [ // ...existing benchmarks... { name: 'My Benchmark', fn: runMyBenchmark }, ];

3. Run and Verify

npm run eval:quick # Check the generated report in docs/evals/
Guidelines

Code Style

Keep these guidelines in mind when contributing.

TypeScript Strict

All code must pass tsc --strict. No any types, no implicit returns, no unused variables. The compiler is your friend.

ESM Only

Pure ECMAScript Modules throughout. Use .js extensions in imports (even for TypeScript files). No require().

No GPL Dependencies

All dependencies must be MIT, Apache-2.0, or BSD licensed. No GPL or LGPL. Check before adding a new dependency.

Test Everything

New features need unit tests. New algorithms need eval benchmarks. If it affects matching quality, it should show up in the eval report.

Important

Never commit real embedding vectors, private keys, or personal data to the repository. Test data should be synthetic or clearly marked as sample data. Eval benchmarks use curated test pairs, not real user data.
