Methodology

How We Evaluate

The eval suite runs real benchmarks with real data and generates timestamped reports in docs/evals/.

Eval Methodology

All benchmarks run against 50 curated need-offer pairs across categories (housing, skills, services, community). Recall is measured as the fraction of known-matching pairs that the system correctly identifies. FPR is measured as the fraction of returned matches that are not in the ground-truth set. Latency metrics use p95 over 100+ iterations. All crypto benchmarks verify round-trip losslessness. No real user data is used—all test data is synthetic.

Running Evals

# Quick eval: ~2 minutes, 10K vectors npm run eval:quick # Full eval: ~15 minutes, 100K vectors npm run eval # Reports saved to docs/evals/eval-YYYY-MM-DD-HH-MM-SS.md
CI Integration

The eval suite runs as part of CI. Any metric that fails its target will fail the build. INFO-level metrics are tracked for trend analysis but do not block merges.
