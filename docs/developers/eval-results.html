Benchmarks

Benchmark Descriptions

The eval suite consists of 9 benchmarks, each testing a different aspect of the system.

Performance

Bootstrap Benchmark

Measures time to initialize the embedding model from cold start. Target: under 30 seconds. Actual: 0.9s on Apple Silicon.

Performance

Embedding Benchmark

Measures embedding latency across 100 representative texts. Reports p50, p95, p99, mean, min, and max. Target: p95 under 100ms.

Privacy

Perturbation Benchmark

Tests Laplace noise at three epsilon levels (5.0, 1.0, 0.1). Verifies that perturbation reduces similarity appropriately and that vectors remain normalized.

Search

HNSW Benchmark

Builds a 10K-vector HNSW index, measures insert time, search latency (p50/p95/p99), and memory usage. Target: search p95 under 5ms.

Quality

Recall Benchmark

Tests matching recall across 50 curated need-offer pairs under different perturbation strategies: baseline, index-only, and double perturbation at multiple thresholds.

Quality

Rewriting Benchmark

Measures the impact of query rewriting on need-offer similarity and recall. Compares raw vs rewritten embeddings.

Architecture

MatchingIndex Benchmark

Tests the separate need/offer index architecture. Compares recall and FPR against a flat single-index approach. Measures same-type noise elimination.

Multilingual

Cross-Language Benchmark

Tests matching across languages (English, Spanish, Catalan). Measures cross-language similarity, same-language non-English similarity, and overall viability.

End-to-End

Latency Benchmark

Measures the full pipeline: embed, perturb, search, match. Target: p95 under 2 seconds. Actual: 29.2ms at the 95th percentile.

Privacy Tradeoff

Privacy vs. Utility

The fundamental tradeoff in differential privacy: more noise means more privacy but less matching accuracy. Here is how each epsilon level performs.

Epsilon Privacy Level Cosine Similarity Recall @0.50 Interpretation
5.0 Low 0.994 94% Minimal noise. Vectors nearly identical to originals. Good for trusted relays.
1.0 Medium 0.880 82% Moderate noise. Signal preserved, real privacy added. Recommended default.
0.1 High 0.179 ~5% Heavy noise. Strong privacy, but very low utility. Requires lower thresholds.
Recommendation

For most deployments, epsilon=1.0 (medium) provides the best balance. It adds enough noise that a relay cannot meaningfully reconstruct the original text, while preserving enough signal for 82% recall. The confirmation step on true embeddings catches the remaining false positives.

Running

How to Run

The eval suite is part of the monorepo and can be run locally.

Quick Eval

Runs all benchmarks with 10K vectors. Takes about 2 minutes on modern hardware.

npm run eval:quick

Full Eval

Runs all benchmarks with 100K vectors for more statistically significant results.

npm run eval

Reports

Reports are auto-generated in Markdown format and saved to docs/evals/ on every run. Each report is timestamped and includes all metrics, analysis, and the privacy vs. utility tradeoff table.

CI Integration

The eval suite runs as part of CI. Any metric that fails its target will fail the build. INFO-level metrics are tracked for trend analysis but do not block merges.
